易宿 (Yisu Hotel) 酒店预订平台 - 技术要点说明

1. 项目架构与技术选型​

本项目严格遵循大作业规范，采用全栈一体化开发模式，分离商户管理端（PC）与用户预订端（移动端）。

  • 核心框架：Next.js 16 (App Router 模式)，无缝结合 React 19（前端界面）与 Node.js（后端 API 服务）。

  • UI 组件库：

  • C 端 (移动端)：使用 antd-mobile 构建原生级丝滑交互体验（如 Popup 弹层、Swiper 轮播、Calendar 日历）。

  • B 端 (PC端)：使用 @ant-design/pro-components (ProLayout, ProTable 等) 快速搭建企业级中后台页面。

  • 样式方案：Tailwind CSS，实现高度定制化的原子级样式与快速响应式布局。

  • 数据库与 ORM：SQLite + Prisma ORM，实现强类型的数据库交互与 Schema 迁移。

2. 核心技术实现要点​

2.1 C 端移动端体验优化 (聚焦用户体验)​

  • 高性能长列表渲染：在“酒店列表页”中，摒弃了传统的页码翻页，利用原生的 IntersectionObserver API 监听 DOM 触底事件，结合后端的 skip 和 take 实现平滑的无限滚动加载 (Infinite Scroll)，极大提升了移动端浏览体验。
  • 状态隔离与防误触设计：在“入住/离店日期选择”的日历组件中，采用“临时状态 (tempDates)”与“真实状态 (dates)”双向绑定的策略。用户在弹窗内的操作不会引起外部界面的抖动，仅在点击“确定”后才完成状态同步，并通过 dayjs 实现了精确的间夜计算与界面定制渲染。
  • 多维联合筛选：实现了基于 LBS 城市定位、价格区间、星级要求及搜索关键字的联合查询过滤逻辑。

2.2 B 端复杂数据与状态管理 (聚焦业务逻辑)​

  • RBAC 权限控制与路由拦截：通过鉴权系统区分 ADMIN (管理员) 与 MERCHANT (商户) 角色。在 layout.tsx 顶层统一进行角色校验，实现越权访问拦截，并动态渲染不同的侧边栏菜单。
  • 级联数据处理：酒店 (Hotel) 与房型 (Room) 存在一对多关系。在商户录入阶段，利用 ProFormList 实现了复杂的嵌套表单提交；后端利用 Prisma 的 include 语法完成多表联查与联级写入。
  • 软删除与状态流转：管理员进行“下线”操作时，并非执行物理删除，而是通过更新 status='OFFLINE' 实现软删除，满足审核留存与随时“恢复上线”的业务需求。

3. 数据库与数据流设计​

项目的数据基石由 Prisma Schema 驱动，具有高度的可扩展性：

  1. User 模型：存储账号、密码加密及角色 (role 字段区分商户与管理员)。
  2. Hotel 模型：包含标题、地址、评分、状态 (PENDING、PUBLISHED、REJECTED 等) 及审核驳回原因。
  3. Room 模型：关联对应的 Hotel ID，记录房型名称、价格及库存量。
  4. 自动化 Seed 脚本：编写了专用的 seed.ts 脚本，可通过一条命令一键生成结构化的初始演示数据（包含商户账号、测试酒店及关联房型），极大提高了团队开发与最终答辩演示的效率。

4. 敏捷开发与团队协作​

项目采用敏捷协作模式，职责划分清晰，确保高质量交付：

  • 技术负责人 (周卫夫)：统筹核心架构，负责底层 Prisma Schema 设计与迁移、鉴权系统及核心 API 路由。
  • B 端研发 (吴隆伟)：负责 PC 端 Admin 系统搭建、商户信息录入表单，实现管理员视角的复杂审核状态流转。
  • C 端研发 (唐雪)：负责移动端整体交互架构，重点攻克首页 Banner、动态日历选择器、高复用组件以及核心的酒店列表无限加载与详情展示。

5. 代码质量与工程化​

  • 严格的 TypeScript 规范：全站应用 TypeScript，消除 any 类型，通过 Prisma 自动生成的类型定义保证了前后端数据传输的安全与一致性。
  • 提交规范与 Lint：集成 Husky、Commitlint 与 ESLint / Prettier，在代码提交前自动进行格式化与质量校验，保证了多人协作的仓库整洁度。
