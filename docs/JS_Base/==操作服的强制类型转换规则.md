# ==操作符的强制类型转换规则

### 1. 字符串和数字比较
当字符串和数字比较时，字符串会被转换为数字。
```js
console.log('123' == 123); // true
console.log('123' == 456); // false
console.log('0' == false); // true
console.log('1' == true); // true
console.log('' == 0); // true 空字符串会被转换为 0 进行比较
console.log('' == false); // true

```

### 2. 布尔值和其他类型比较
当布尔值和其他类型比较时，布尔值会被转换为数字。
```js
console.log(true == 1); // true
console.log(false == 0); // true
console.log(true == 2); // false
```

### 3. null和undefined比较
null和undefined是相等的。仅等于自身和对方
当null和undefined比较时，它们会被转换为数字0。
```js
console.log(null == undefined); // true
console.log(null == null); // true
console.log(undefined == undefined); // true
console.log(null == 0); // false
console.log(undefined == 0); // false
```

### 4. 对象和非对象比较
当对象和非对象比较时，对象会被转换为原始值。
```js
console.log([1, 2] == '1,2'); // true
console.log([1] == 1); // true
console.log({} == '[object Object]'); // true
```

### 5. 原始值和原始值比较
当原始值和原始值比较时，它们会直接比较。

### 6. 严格相等运算符（===）
严格相等运算符不会进行类型转换，只有在类型和值都相等时才返回true。

### 7.符号和其他类型比较
Symbol类型只能与Symbol类型进行比较，与其他类型的比较总是返回false。
```js
console.log(Symbol('123') == 'Symbol(123)'); // true
```
