酒店列表的展示上，主要采用了**客户端渲染（CSR）结合原生浏览器 API 实现的无限滚动**机制。

具体的技术实现和逻辑分析如下：

* **状态驱动渲染**：列表的核心数据流由 React 的 `useState` 管理，包括酒店数组（`hotels`）、当前页码（`page`）、是否还有更多数据（`hasMore`）以及加载状态（`isLoading`）。每次状态更新都会触发界面的重新渲染。
* **触底加载（无限滚动）的核心技术**：项目没有依赖第三方的滚动或分页组件，而是巧妙地使用了浏览器原生的 **`IntersectionObserver` API**。
    * 其逻辑是在列表的最底部放置了一个带有 `ref`（`observerRef`）的 `div` 元素作为“探测器”。
    * 当用户向下滑动，这个底部元素进入屏幕可视区域（`isIntersecting` 为 true）且满足还有更多数据且当前未在加载中时，就会触发页码 `page` 加 1 的操作。
* **数据拉取与拼接**：页码的改变会触发 `useEffect` 钩子内的 `fetchHotels` 函数。使用原生 `fetch` 向后端的 `/api/public/hotel` 接口发起请求。如果是第一页则直接替换数据，如果是后续页码，则将新拉取的数据追加（`[...prev, ...result.data]`）到现有的 `hotels` 数组中。
* **UI 与样式构建**：列表的 UI 完全是通过原生的 HTML 标签（`div`, `img`, `h3` 等）结合 **Tailwind CSS** 的工具类搭建的。例如使用了 `sticky` 实现顶部筛选栏的吸顶效果，使用 `line-clamp-1` 实现标题的单行截断。
* **路由参数的异步边界**：因为组件使用了 Next.js App Router 的 `useSearchParams` 来获取首页传来的城市和搜索关键字参数，根据 Next.js 的规范，包含此 Hook 的组件被强制包裹在了 React 的 `<Suspense>` 组件内，以提供流式渲染的回退 UI（即“加载中...”提示）。
