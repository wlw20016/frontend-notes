## 设计模式
在面型对象软件 设计过程中，针对特定问题的简洁的解决方案

## 工厂模式
在 JavaScript 中，**工厂模式（Factory Pattern）**是最常用的**创建型设计模式**之一。它的核心思想是：**不暴露创建对象的具体逻辑，而是将这部分逻辑封装在一个专门的函数中**。这个专门负责“生产”对象的函数，就被称为“工厂”。

你可以把它想象成现实生活中的汽车制造厂：你不需要知道汽车的具体组装细节（如怎么拧螺丝、装轮胎），你只需要向工厂提出需求（比如“我要一辆红色的轿车”），工厂就会把造好的车交给你。



---

### 1. 为什么需要工厂模式？

在没有工厂模式时，如果我们想创建多个具有相似属性的对象，往往会写出很多重复的代码：

```javascript
let person1 = {
    name: "Alice",
    age: 25,
    sayName: function() { console.log(this.name); }
};

let person2 = {
    name: "Bob",
    age: 30,
    sayName: function() { console.log(this.name); }
};
// 如果有 100 个人，就要写 100 次几乎一样的代码
```

工厂模式的出现就是为了**解决创建多个相似对象时的代码冗余问题**。

---

### 2. 经典的 JS 工厂模式实现

通过封装一个函数，接收参数并返回一个包含特定属性和方法的对象：

```javascript
// 这就是一个“工厂”函数
function createPerson(name, age, job) {
    let obj = new Object(); // 1. 创建一个新对象
    
    // 2. 为对象添加属性和方法
    obj.name = name;
    obj.age = age;
    obj.job = job;
    obj.sayHello = function() {
        console.log(`你好，我是 ${this.name}，我的职业是 ${this.job}。`);
    };
    
    // 3. 返回这个对象
    return obj;
}

// 使用工厂函数批量生产对象
let person1 = createPerson("张三", 28, "前端工程师");
let person2 = createPerson("李四", 35, "产品经理");

person1.sayHello(); // 输出: 你好，我是 张三，我的职业是 前端工程师。
person2.sayHello(); // 输出: 你好，我是 李四，我的职业是 产品经理。
```

### 3. ES6 Class 时代的工厂模式

随着 ES6 引入了 `class`，工厂模式常常演变成一个专门用来实例化不同类的静态方法。这在处理复杂对象结构时非常有用：

```javascript
class Dog {
    speak() { console.log("汪汪汪！"); }
}

class Cat {
    speak() { console.log("喵喵喵！"); }
}

// 动物工厂
class AnimalFactory {
    static createAnimal(type) {
        switch(type) {
            case 'dog':
                return new Dog();
            case 'cat':
                return new Cat();
            default:
                throw new Error("工厂不能生产这种动物");
        }
    }
}

let myDog = AnimalFactory.createAnimal('dog');
myDog.speak(); // 输出: 汪汪汪！
```

---

### 4. 工厂模式的优缺点

#### **优点：**
* **消除重复代码：** 集中管理对象的创建过程，避免在业务代码中到处写 `new Object()` 或字面量。
* **解耦：** 调用者不需要知道对象创建的具体细节，只需要知道如何使用即可。如果对象的构造逻辑发生变化，只需要修改工厂函数内部，不需要修改调用它的地方。

#### **缺点：**
* **对象识别问题（针对经典函数式工厂）：** 使用传统的 `createPerson` 创建的对象，其构造函数统一指向最顶层的 `Object`，导致你无法通过 `instanceof` 来判断这个对象究竟是什么具体类型（比如你无法区分它是 Person 还是 Animal）。
    ```javascript
    console.log(person1 instanceof Object); // true
    // 无法判断 person1 是一个 Person 类型的实例
    ```
* **内存开销（针对经典函数式工厂）：** 每次调用工厂函数创建新对象时，对象内部的方法（如 `sayHello`）都会被重新创建一遍，这会造成额外的内存浪费（这个问题后来被**构造函数模式**和**原型模式**解决了）。

## 单例模式
调用方法，获取单例对象，重复调用获取的是相同对象
```js
//1.定义类
class SingleTon{
    //2.添加静态属性
    static #instance

    //3.添加静态方法
    static getInstance(){
        //4.判断并返回对象
        if( this.#instance === undefined){
            this.#instance = new SingleTon()
        }
        return this.#instance
    }
}
```
## 观察者模式
多个观察对象订阅一个目标对象，目标对象状态改变的时候，所有观察对象会收到通知
