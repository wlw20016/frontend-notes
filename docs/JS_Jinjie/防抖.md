实现一个防抖（Debounce）函数是前端开发中非常常见的需求。防抖的核心思想是：**无论事件触发得多么频繁，只有在停止触发 n 秒后，才真正执行目标函数。如果在这 n 秒内又触发了事件，则重新计时。**

这在处理如窗口大小调整（`resize`）、搜索框输入（`keyup` / `input`）或按钮频繁点击时非常有用，可以有效减少性能损耗。

下面为你提供一个功能完善的 JavaScript 防抖函数实现，包含基础版本和支持“立即执行”的高级版本。

### 标准防抖函数实现

这个版本包含了对 `this` 指向和参数 `arguments` 的正确处理，并且支持一个 `immediate` 参数，允许你选择是在触发开始时立即执行，还是在等待时间结束后执行。

```javascript
/**
 * 防抖函数
 * @param {Function} func - 需要防抖的源函数
 * @param {number} wait - 等待时间（毫秒）
 * @param {boolean} immediate - 是否立即执行（true: 触发时立即执行一次，然后等待 wait 毫秒；false: 等待 wait 毫秒后执行）
 * @returns {Function} - 返回一个新的包装函数
 */
function debounce(func, wait, immediate = false) {
  let timeout = null;

  // 返回一个闭包函数
  return function (...args) {
    // 保存当前的 this 上下文，确保源函数能在正确的环境中执行
    const context = this;

    // 每次触发时，清除之前的定时器，重新计时
    if (timeout) clearTimeout(timeout);

    if (immediate) {
      // 如果要求立即执行，且当前没有定时器在等待中，则执行函数
      const callNow = !timeout;
      
      // 设置定时器，在 wait 时间后清空 timeout，允许下一次立即执行
      timeout = setTimeout(() => {
        timeout = null;
      }, wait);
      
      if (callNow) {
        func.apply(context, args);
      }
    } else {
      // 如果不要求立即执行，则单纯设置定时器延迟执行
      timeout = setTimeout(() => {
        func.apply(context, args);
      }, wait);
    }
  };
}
```

### 如何使用？

以下是一个结合实际场景（例如搜索框输入）的使用示例：

```javascript
// 1. 假设这是你的核心搜索逻辑（非常消耗资源）
function search(keyword) {
  console.log(`正在请求接口搜索关键字: ${keyword}`);
}

// 2. 使用 debounce 包装这个函数，设置 500ms 的延迟
const debouncedSearch = debounce(search, 500);

// 3. 绑定到输入框的 input 事件上
// 假设 HTML 中有一个 <input id="searchInput" />
const inputEl = document.getElementById('searchInput');

inputEl.addEventListener('input', function(event) {
  // 用户连续快速输入时，不会立刻调用 search 函数
  // 只有当用户停下键盘输入 500ms 后，才会打印上面的 console.log
  debouncedSearch(event.target.value); 
});
```

### 核心原理解析

* **闭包（Closure）：** 外层的 `debounce` 函数执行后返回了内层函数，内层函数维持了对 `timeout` 变量的引用，这样每次触发事件时，都能访问并修改同一个定时器标识。
* **清理定时器（`clearTimeout`）：** 这是防抖的灵魂。只要事件被再次触发，就立马撤销上一次的定时任务，重新开始倒计时。
* **上下文与参数（`this` 和 `args`）：** 在 JavaScript 的事件回调中，`this` 通常指向触发事件的 DOM 元素，且会自带一个 `Event` 对象作为参数。使用 `func.apply(context, args)` 可以完美地将这些环境信息透传给真正的目标函数。
