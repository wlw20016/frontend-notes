继承方式

JavaScript 的继承机制在底层是基于**原型链（Prototype Chain）**的。随着语言的发展，开发者们探索出了多种实现继承的方式，直到 ES6 推出了官方的语法糖。

以下是 JS 中常见的 7 种继承方式，按照演进顺序整理：

1. 原型链继承 (Prototype Chain Inheritance)​

这是最基础的继承方式，通过将子类的原型指向父类的实例来实现。

function Parent() {
  this.name = 'Parent';
  // 引用类型属性
  this.colors = ['red', 'blue', 'green'];
}

Parent.prototype.getName = function() {
  console.log(this.name);
};

function Child() {}

// 核心：子类原型指向父类实例
Child.prototype = new Parent();

let child1 = new Child();
let child2 = new Child();

child1.colors.push('black');

// 缺点暴露：引用类型被所有实例共享
console.log(child1.colors); // ['red', 'blue', 'green', 'black']
console.log(child2.colors); // ['red', 'blue', 'green', 'black']
  • 实现： Child.prototype = new Parent();
  • 优点： 写法简单，子类实例可以访问父类原型上的方法。
  • 缺点： * 父类中的引用类型属性（如数组、对象）会被所有子类实例共享，修改一个会影响所有。
    • 创建子类实例时，无法向父类的构造函数传递参数。

2. 借用构造函数继承 (Constructor Stealing)​

为了解决原型链继承中引用类型共享和传参的问题，在子类构造函数中调用父类构造函数。

  • 实现： 核心原理： 在子类构造函数内部调用父类构造函数。
function Parent(name) {
  this.name = name;
  this.colors = ['red', 'blue', 'green'];
}

Parent.prototype.getName = function() {
  console.log(this.name);
};

function Child(name, age) {
  // 核心：借用父类构造函数，并传递参数
  Parent.call(this, name); 
  this.age = age;
}

let child1 = new Child('Tom', 18);
let child2 = new Child('Jerry', 20);

child1.colors.push('black');

// 优点体现：引用类型不共享了，且可以传参
console.log(child1.colors); // ['red', 'blue', 'green', 'black']
console.log(child2.colors); // ['red', 'blue', 'green']
console.log(child1.name);   // Tom

// 缺点暴露：无法访问父类原型上的方法
// child1.getName(); // 报错：TypeError: child1.getName is not a function
  • 优点： 解决了引用类型共享问题，且可以向父类传参。
  • 缺点： 方法必须都在构造函数中定义，无法实现函数的复用；且子类无法访问 Parent.prototype 上定义的方法。

3. 组合继承 (Combination Inheritance)​

结合了原型链和构造函数，是 ES5 中最常用的继承模式。

  • 实现： 核心原理： 原型链继承（拿方法） + 借用构造函数（拿属性并解决共享问题）。
function Parent(name) {
  this.name = name;
  this.colors = ['red', 'blue', 'green'];
}

Parent.prototype.getName = function() {
  console.log(this.name);
};

function Child(name, age) {
  // 第 1 次调用父类：继承实例属性
  Parent.call(this, name); 
  this.age = age;
}

// 第 2 次调用父类：继承父类原型上的方法
Child.prototype = new Parent(); 
// 修复构造函数指向
Child.prototype.constructor = Child; 

let child1 = new Child('Tom', 18);
let child2 = new Child('Jerry', 20);

child1.colors.push('black');

// 既能复用方法，又保证了属性的独立
console.log(child1.colors); // ['red', 'blue', 'green', 'black']
console.log(child2.colors); // ['red', 'blue', 'green']
child1.getName(); // Tom
child2.getName(); // Jerry

// 缺点：Parent 被调用了两次，子类的原型上（Child.prototype）多了一份没用的 name 和 colors 属性。
  • 优点： 既保证了原型上方法的复用，又保证了每个实例有自己的属性。
  • 缺点： 最大的缺点是调用了两次父类的构造函数，导致子类的原型上多了不必要的父类属性（被实例属性屏蔽了），造成了一定程度的内存浪费。

4. 原型式继承 (Prototypal Inheritance)​

借助 Object.create() 方法实现（在 ES5 之前开发者通常用一个空函数做中间介质）。适用于不需要显式创建构造函数，只想基于某个对象生成新对象的场景。

  • 实现： 核心原理： 基于已有的对象创建一个新对象。
let parentObj = {
  name: 'Parent',
  colors: ['red', 'blue', 'green']
};

// 核心：使用 Object.create 基于 parentObj 创建新对象
let child1 = Object.create(parentObj);
let child2 = Object.create(parentObj);

child1.name = 'Tom'; // 给 child1 添加实例属性，屏蔽了原型上的 name
child1.colors.push('black');

// 缺点同原型链继承：引用类型被共享
console.log(child1.name);   // Tom
console.log(child2.name);   // Parent
console.log(child1.colors); // ['red', 'blue', 'green', 'black']
console.log(child2.colors); // ['red', 'blue', 'green', 'black']
  • 优点： 无需创建自定义类型即可实现继承。
  • 缺点： 和原型链继承一样，引用类型的属性会被所有实例共享。

5. 寄生式继承 (Parasitic Inheritance)​

在原型式继承的基础上，在函数内部增强对象，最后返回这个对象。

  • 实现： 核心原理： 创建一个仅用于封装继承过程的函数，在内部增强对象。
let parentObj = {
  name: 'Parent',
  colors: ['red', 'blue', 'green']
};

function createChild(original) {
  let clone = Object.create(original); // 创建新对象
  // 核心：增强对象
  clone.sayHi = function() {
    console.log('Hi, my name is ' + this.name);
  };
  return clone;
}

let child1 = createChild(parentObj);
child1.name = 'Tom';
child1.sayHi(); // Hi, my name is Tom

// 缺点：每次调用 createChild 都会重新创建 sayHi 方法，无法复用函数。
  • 优点： 封装了继承的过程，可以为新对象添加特定的方法。
  • 缺点： 和借用构造函数一样，每次创建对象都会创建一遍方法，无法复用。

6. 寄生组合式继承 (Parasitic Combination Inheritance)​

这是 ES5 中最完美的继承方式。它通过借用构造函数来继承属性，通过原型链的混成形式来继承方法，只调用了一次父类构造函数。

  • 实现： 核心原理： 借用构造函数继承属性，借用 Object.create() 继承原型链。ES5 的最佳实践。
function Parent(name) {
  this.name = name;
  this.colors = ['red', 'blue', 'green'];
}

Parent.prototype.getName = function() {
  console.log(this.name);
};

function Child(name, age) {
  Parent.call(this, name); // 继承属性
  this.age = age;
}

// 核心：不使用 new Parent()，而是创建父类原型的一个副本
Child.prototype = Object.create(Parent.prototype);
// 修复 constructor 指向
Child.prototype.constructor = Child;

let child1 = new Child('Tom', 18);
let child2 = new Child('Jerry', 20);

child1.colors.push('black');

// 完美：方法复用了，属性独立了，且 Parent 构造函数只执行了一次！
console.log(child1.colors); // ['red', 'blue', 'green', 'black']
console.log(child2.colors); // ['red', 'blue', 'green']
child1.getName(); // Tom
  • 优点： 高效（只调用一次父类构造函数），原型链保持不变，能正常使用 instanceof 和 isPrototypeOf()。

7. ES6 类继承 (ES6 class and extends)​

ES6 引入了 class 关键字，这实际上是寄生组合式继承的语法糖，让 JS 的继承写起来更像传统的面向对象语言（如 Java）。

  • 实现： 核心原理： class 和 extends 语法糖，底层依然是寄生组合式继承。
class Parent {
  constructor(name) {
    this.name = name;
    this.colors = ['red', 'blue', 'green'];
  }
  
  // 相当于定义在 Parent.prototype 上
  getName() {
    console.log(this.name);
  }
}

class Child extends Parent {
  constructor(name, age) {
    // 核心：必须在 constructor 中优先调用 super()
    super(name); 
    this.age = age;
  }
  
  getAge() {
    console.log(this.age);
  }
}

let child1 = new Child('Tom', 18);
let child2 = new Child('Jerry', 20);

child1.colors.push('black');

console.log(child1.colors); // ['red', 'blue', 'green', 'black']
console.log(child2.colors); // ['red', 'blue', 'green']
child1.getName(); // Tom
child1.getAge();  // 18
  • 优点： 语法简洁明了，标准化，是现代前端开发的绝对主流。

总结来说： 在现代开发中，你几乎只需要使用 ES6 的 class extends 就可以了。但理解前面的几种方式（尤其是寄生组合式继承）对于深入理解 JavaScript 的原型机制（Prototype）和应对技术面试是非常必要的。
