# 原型原型链
## 构造函数中，会存在浪费内存的现象
```js
function Star(uname, age){
    this.uname = uname,
    this.age = age,
    this.sing = ()=>{
        console.log('唱歌ing');
    }
}
//这时，每new一个实例，就会额外为sing()函数开辟一个空间，造成内存的浪费
```

# 原型
## 什么是原型
JS规定，每一个**构造函数**都有一个prototype属性，指向另一个对象，所以我们也称为原型对象。也就是，prototype是一个指针，指向一个对象
这个对象可以挂载函数，可以把不变的方法，定义在prototype对象上，这样所有对象的实例就可以共享这些方法。 构造函数和原型对象中的this都指向实例化的对象。（通过实例调用原型对象上的方法时，根据 JavaScript 的隐式绑定规则，原型对象中的函数的 this 指向调用该方法的实例对象。） 

## constructor属性
原型对象中都有的属性。 constructor(构造器)，是指向该原型对象的构造函数的。
这有什么用呢？   
我们可以通过constructor属性，来判断一个对象，到底是哪个构造函数创建的。

# 对象原型
## 当你使用 new 关键字调用构造函数时，JavaScript 内部会经历以下步骤：
```
1.创建一个全新的对象。
2.将这个新对象的原型(__proto__)指向构造函数的 prototype 属性。 __proto__ = prototype 现代浏览器对**对象原型**标准的写法是：[[prototype]]
3.将构造函数内部的 this 绑定到这个新创建的对象上。
4.执行构造函数内部代码（通过 this 给新对象添加属性）。
5.返回这个新对象。
```
对象原型(__proto__)也有一个constructor属性，指向该原型对象的构造函数。

# 原型继承
通过原型来继承
```js
function Person(){}
function Women(){}
Women.prototype = new Person() 
//这里就相当于是Women的原型对象里的__proto__指向的是Person的原型对象，因为我们是实例化了Person来创建Women的原型对象的，实例化的Person对象中的__proto__就是指向Person的原型对象的
Women.prototype.constructor = Women

```
