这是一个非常经典的 JavaScript 基础问题。我们可以用一个表格和几条核心建议来简单、清晰地总结它们：

### 核心区别对比

| 特性 | `var` | `let` | `const` |
| :--- | :--- | :--- | :--- |
| **作用域** | **函数级**或全局级 | **块级** (`{}`) | **块级** (`{}`) |
| **变量提升 (Hoisting)** | 提升，且初始化为 `undefined` | 提升，但存在**临时性死区 (TDZ)** | 提升，但存在**临时性死区 (TDZ)** |
| **同作用域重复声明** | 允许 | **不允许** | **不允许** |
| **重新赋值** | 允许 | 允许 | **不允许** (声明时必须初始化) |
| **绑定到全局对象** (如浏览器中的 `window`) | 是 | 否 | 否 |

---

### 详细特性总结

* **`var` (传统的声明方式)**
    * **缺点多**：var只有函数级作用域和全局作用域。容易造成变量污染（全局变量泄露）、因为允许重复声明导致意外覆盖变量、因为没有块级作用域在 `if`, `for` 语句块中容易产生经典的闭包陷阱。
    * **现状**：在现代 JavaScript 开发中，**强烈建议彻底放弃使用 `var`**。

* **`let` (现代的变量声明)**
    * **定位**：用于声明**会被修改**的变量（比如循环计数器、状态标志等）。
    * **优点**：引入了块级作用域和临时性死区，代码行为更可预测，避免了 `var` 的大部分缺陷。

* **`const` (现代的常量声明)**
    * **定位**：用于声明**不会被重新赋值**的变量。
    * **注意点**：`const` 保证的是**变量指向的内存地址不变**。如果是基本类型（如数字、字符串），它的值就完全不可变；但如果是引用类型（如对象、数组），你**仍然可以修改对象内部的属性或数组的元素**。
