简单来说，**懒加载（Lazy Loading）** 是一种前端性能优化的策略。它的核心思想就是：**“按需加载”**。

与其在页面一打开时就一次性加载所有资源（图片、视频、脚本等），不如等用户真正需要看到它们时（比如滚动页面到相应位置），再开始加载。

-----

### 1\. 为什么要用懒加载？

想象一下你正在刷一个有 100 张高清图的网页：

  * **不使用懒加载：** 浏览器会尝试同时下载这 100 张图。这会导致网页首屏加载极慢，浪费用户的流量（万一用户只看了前两张就关掉了呢？），甚至导致页面卡死。
  * **使用懒加载：** 页面打开时只加载前 3 张。剩下的 97 张，只有当你向下滚动、快要看到它们时，浏览器才会去下载。

### 2\. 核心工作原理

懒加载通常遵循以下步骤：

1.  **占位：** 图片的 `src` 属性先不放真实的图片链接，而是放一个极小的占位图或者干脆留空。真实的链接存在一个自定义属性里（如 `data-src`）。
2.  **监听：** 通过 JavaScript 监听页面的滚动，或者使用现代浏览器自带的 `Intersection Observer` API。
3.  **替换：** 当图片进入（或即将进入）浏览器的可视区域时，脚本将 `data-src` 的值赋给 `src`。
4.  **展示：** 浏览器发现 `src` 改变，自动发起请求下载并显示图片。

-----

### 3\. 懒加载的优点

  * **提升加载速度：** 减少了首屏需要下载的资源总量，让网页瞬间打开。
  * **节省流量：** 只下载用户实际看过的内容。
  * **减轻服务器压力：** 避免了短时间内并发大量的 HTTP 请求。

### 4\. 现在的主流做法

以前实现懒加载需要写复杂的 JavaScript，但现在非常简单：

  * **原生 HTML 支持：** 现代浏览器直接支持 `loading="lazy"` 属性。
    ```html
    <img src="real-image.jpg" loading="lazy" alt="示例图片">
    ```
  * **框架支持：** 在 React, Vue 或 Next.js 等框架中，都有成熟的插件（如 `vue-lazyload`）或内置组件来处理更复杂的懒加载逻辑。

-----

**总结：** 懒加载就是一种“拖延症”策略，虽然在生活中不提倡，但在网页开发中，它能显著提升用户体验。

1. 组件级别的按需懒加载 (Component-level Lazy Loading)
在你的移动端首页文件 src/app/mobile/index.tsx 中，使用了 Next.js 提供的 next/dynamic 来引入城市选择器组件。代码实现为：
 ```js
const CitySelector = dynamic(() => import('./components/CitySelector'), { ssr: false })
 ```


架构分析：这是一个非常经典的 Code Splitting（代码分割）策略。因为城市选择器内部通常包含了庞大的城市数据集合（如拼音索引、数百个城市列表等），如果在首屏直接将其打包进去，会严重拖慢首页的初始加载速度。通过 `dynamic` 配合 `{ ssr: false }`，使得这部分庞大的代码块只有在组件真正需要渲染（也就是用户点击触发弹出时）才会在客户端发起请求拉取，极大地优化了首屏的 JS Bundle 体积。
