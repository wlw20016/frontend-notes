### 一、 核心亮点撰写策略与“钓鱼钩”设计

#### 1. 渲染性能与长列表架构 (涵盖：虚拟列表、懒加载)
* **简历描述**：主导移动端首页长列表渲染性能优化。针对海量酒店卡片导致的 DOM 节点溢出与滑动掉帧问题，废弃传统的分页渲染，基于 `React Virtual` 落地动态高度虚拟列表。结合 `IntersectionObserver` 与 `next/image` 实现视口外图片的按需懒加载，将列表长距离滑动帧率（FPS）稳定在流畅级别，彻底解决低端机型的白屏与内存泄漏问题。
* **🎣 面试官必定追问**：
    * *“你们的酒店卡片高度是固定的还是动态的？动态高度的虚拟列表在快速滑动时，你是怎么避免滚动条乱跳的？”*
    * *“懒加载的图片如果加载失败或者过慢，你做了什么兜底策略？”*

#### 2. 核心交易链路与高并发交互保护 (涵盖：状态锁、防抖)
* **简历描述**：重构 C 端酒店预订与高频搜索的核心交互链路。针对搜索框高频触发的性能瓶颈，自主封装泛型防抖（Debounce）Hook 降低接口调用频率；针对核心交易按钮（如“去预订”），摒弃常规的定时器防抖，设计并实现了基于数据 ID 的“局部状态锁”与全局防点透遮罩（Mask Overlay）。在保证页面级组件作用域隔离的同时，从物理层面彻底杜绝了网络拥堵下的重复提交与路由“幽灵跳转”漏洞。
* **🎣 面试官必定追问**：
    * *“你简历里强调了‘摒弃常规的定时器防抖’，能说说防抖和状态锁在处理按钮点击时，底层逻辑到底有什么区别吗？为什么交易链路必须用锁？”*
    * *“如果一个请求发出去之后，后端响应超级慢，用户强制刷新了页面，你的状态锁还有效吗？”*

#### 3. 企业级数据安全与鉴权架构 (涵盖：完整鉴权)
* **简历描述**：设计并重构后台管理系统（B端）的安全鉴权体系。废弃存在 XSS 隐患的 LocalStorage 存储方案，引入 JWT (JSON Web Token) 与 HttpOnly Cookie 机制保障凭证安全。基于 Next.js Edge Runtime 编写全局 `proxy` 中间件，实现 API 路由与页面路由的双重无缝拦截，彻底修复了商户数据越权访问（IDOR）与伪造身份漏洞。
* **🎣 面试官必定追问**：
    * *“既然你把 Token 放在了 HttpOnly Cookie 里，那前端代码怎么知道当前用户的角色和 ID 呢？”*
    * *“中间件拦截和传统的在组件 `useEffect` 里判断登录状态，在页面渲染的生命周期上有什么不同？”*

#### 4. 前端工程化、数据监控与底层基建 (涵盖：埋点、代码规范、部署)
* **简历描述**：从零搭建前端无侵入式埋点监控 SDK（覆盖 PV、点击、组件曝光）。利用 `navigator.sendBeacon` 实现页面卸载时的安全数据上报；通过抽取高复用性的独立组件与 Hook 隔离埋点逻辑，严格遵守 TypeScript 规范（零 `any`），保证业务代码的纯净度。此外，熟练使用 GitHub Actions 结合 GitHub Pages 构建自动化部署工作流，沉淀项目架构分析与在线技术文档。同时，具备扎实的 JavaScript 底层功底与算法思维，能运用最优的时间/空间复杂度处理前端复杂数据状态。
